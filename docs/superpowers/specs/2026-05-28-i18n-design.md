# 多语言改造设计 — if_mapping_cap + if_mapping_gui

**日期**: 2026-05-28
**状态**: 待实现
**目标语言**: 中文（zh）、日语（ja）
**涉及项目**: `if_mapping_cap`（CAP TypeScript 后端）、`if_mapping_gui`（Python customtkinter GUI）

---

## 1. 需求摘要

两个项目当前 UI 文本/错误消息均为硬编码：
- `if_mapping_gui`：界面文本全部为中文，无任何 i18n 基础设施
- `if_mapping_cap`：错误消息为英文，面向用户的日志消息也为英文

目标：支持中文和日语两种界面语言。GUI 通过设置页面切换，CAP 后端根据请求的 `language` 参数动态返回对应语言的错误消息和用户可见日志。

### 2.2 i18n.py 模块

```python
# i18n.py
import json
from pathlib import Path

_FALLBACK = "zh"
_cache: dict[str, dict[str, str]] = {}
_lang = _FALLBACK

def load(lang: str) -> None:
    global _lang
    _lang = lang
    if lang not in _cache:
        path = Path(__file__).parent / "i18n" / f"{lang}.json"
        _cache[lang] = json.loads(path.read_text(encoding="utf-8"))

def t(key: str, **kwargs) -> str:
    strings = _cache.get(_lang) or _cache.get(_FALLBACK) or {}
    text = strings.get(key, key)
    return text.format(**kwargs) if kwargs else text

def current() -> str:
    return _lang
```

### 2.3 App 生命周期集成

- `App.__init__` 读取 `config.ui_language`（默认 `zh`），调用 `i18n.load(lang)`
- `App` 新增 `retranslate()` 方法：调用所有 frame 的 `retranslate()` 方法刷新文本
- `BaseFrame` 新增抽象方法签名 `retranslate()`（默认空实现，各 frame 按需覆盖）

### 2.4 各 Frame 改造方式

每个 frame 的 `_build()` 中：
- 所有 `text=` 参数改为 `text=t("key")`
- 所有硬编码字符串（日志消息、对话框标题等）改为 `t("key", ...)`

新增 `retranslate()` 方法，对所有**静态标签和按钮**调用 `widget.configure(text=t("key"))`。动态文本（日志行、结果摘要）已在生成时调用 `t()`，无需单独处理。

### 2.5 设置页面新增 UI 语言选项

在「默认语言」下方新增：
```
界面语言: [zh ▼]   (选项: zh, ja)
```

保存时：
1. 写入 `config.ui_language`
2. 调用 `i18n.load(new_lang)`
3. 调用 `app.retranslate()`

### 2.6 config.py 变更

`EXCEL_DEFAULTS` 和 `load()`/`save()` 无需改动，仅在 `load()` 返回值中保证 `ui_language` 键存在（默认 `"zh"`）。

### 3.2 srv/utils/i18n.ts 模块

```typescript
import { readFileSync } from 'fs';
import { resolve, dirname } from 'path';
import { fileURLToPath } from 'url';

type Messages = Record<string, string>;

const __dir = dirname(fileURLToPath(import.meta.url));
const _cache = new Map<string, Messages>();

function load(lang: string): Messages {
  if (_cache.has(lang)) return _cache.get(lang)!;
  const file = resolve(__dir, `../i18n/messages_${lang}.json`);
  try {
    const msgs = JSON.parse(readFileSync(file, 'utf-8')) as Messages;
    _cache.set(lang, msgs);
    return msgs;
  } catch {
    // fallback to zh
    if (lang !== 'zh') return load('zh');
    return {};
  }
}

export function t(key: string, lang: string, vars?: Record<string, string>): string {
  const msgs = load(lang);
  let text = msgs[key] ?? key;
  if (vars) {
    for (const [k, v] of Object.entries(vars)) {
      text = text.replaceAll(`{${k}}`, v);
    }
  }
  return text;
}
```

### 3.3 Errors 工厂函数改造

`srv/utils/errors.ts` 的 `Errors` 对象改为接受 `lang` 参数：

```typescript
export const Errors = {
  HANA_CONNECTION:  (lang = 'ja'): AppError =>
    new AppError(t('error.hana_connection', lang), 503),
  AI_CORE_AUTH:     (lang = 'ja'): AppError =>
    new AppError(t('error.ai_core_auth', lang), 502),
  AI_CORE_TIMEOUT:  (lang = 'ja'): AppError =>
    new AppError(t('error.ai_core_timeout', lang), 504),
  INVALID_PROVIDER: (p: string, lang = 'ja'): AppError =>
    new AppError(t('error.invalid_provider', lang, { p }), 400),
  PROMPT_NOT_FOUND: (step: string, promptLang: string, uiLang = 'ja'): AppError =>
    new AppError(t('error.prompt_not_found', uiLang, { step, lang: promptLang }), 500),
};
```

调用方（`if-mapping-service.ts` 等）在调用 `Errors.xxx()` 时传入 `config.language`。

## 4. 翻译键命名约定

- 格式：`<模块>.<描述>`，全小写，下划线分隔
- 占位符：`{name}`，与 Python `str.format()` 和 TypeScript `.replaceAll()` 兼容
- 不翻译：技术术语（Provider、Step、Token、SSE）、列名（CAP 后端内部）
